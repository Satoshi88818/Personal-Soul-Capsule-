SoulCapsule v3 — Full Codebase

A Personal Multimodal Memory Vault. Simple. Beautiful. Alive.

What's new in v3:

rebuild-index command — reconstruct FAISS + ID_MAP from SQLite at any time

delete command — remove a memory by ID, with automatic index rebuild

--import flag — copies media into a managed soulcapsule_media/ folder (content-addressed, no duplicates)

Streamlit web UI — mobile-friendly interface alongside the rich TUI

Auto-rebuild on startup — if the index is missing but the DB exists, rebuilds automatically

Content-addressed media storage — files are hashed on import; duplicates are deduplicated

What Changed in v3

1 — rebuild-index Command

Reads all embeddings stored in SQLite and rebuilds both the FAISS index and the ID_MAP pickle from scratch. Safe to run any time. Also called automatically on startup if the index file is missing but the database has memories.

2 — delete Command

Removes a memory from SQLite by full UUID or unambiguous short prefix (e.g. first 8 chars). After deletion, automatically calls rebuild_index() so the FAISS index is always in sync. This is cleaner than trying to remove individual vectors from IndexFlatL2, which doesn't support it natively.

3 — --import Flag (Self-Contained Vault)

When --import is passed, media files are copied into soulcapsule_media/ before embedding. Files are content-addressed using SHA-256 — identical files are stored only once. The database then stores the managed path, not the original. Moving or archiving the vault folder keeps all media intact.

4 — Streamlit Web UI

A mobile-friendly browser interface over the same core functions. Five tabs: Add, Search, Surprise, Mood drift (rendered with Plotly instead of the terminal sparkline), and a Manage tab with a memory table and delete functionality. The rich TUI remains the primary interface; this is a thin wrapper for on-the-go access.
