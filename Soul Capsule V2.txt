What Changed and Why

1 — Proper Audio Embedding (Whisper)

Before: Audio was never actually analysed. The code created a fake text string like "audio voice note calm short" and embedded that, making audio semantically meaningless.

After: Whisper transcribes the audio to real text first. Acoustic features (loudness, duration) are appended as context tags, then the enriched transcript is embedded. The transcript is also stored in the database so it appears in search results.

# The key fix result = model.transcribe(audio_path) transcript = result["text"].strip() enriched = f"{transcript} [audio: {energy_tag}, {length_tag}, {duration_s:.0f}s]" embedding = embed_text(enriched) 

2 — Proper CLIP Dimension Projection

Before: CLIP produces 512-dim vectors. The code resized them to 384 by truncating — losing the last 128 dimensions entirely, which breaks the geometric meaning of the vector space.

After: A learned orthogonal linear projection layer maps 512 → 384 properly, preserving angular relationships.

proj = torch.nn.Linear(CONFIG["clip_dim"], CONFIG["dimension"], bias=False) torch.nn.init.orthogonal_(proj.weight) projected = projector(clip_vec) # shape: [1, 384] 

3 — Stable FAISS Index Alignment

Before: Search used FAISS vector positions to derive SQLite rowid directly (rowid = faiss_pos + 1). This breaks the moment any memory is deleted, causing every subsequent lookup to return the wrong memory or nothing.

After: An explicit ID_MAP (a list of UUIDs) is persisted alongside the FAISS index. Position i in the FAISS index always maps to ID_MAP[i], which is then looked up by primary key in SQLite.

# On add: INDEX.add(np.array([final_emb])) ID_MAP.append(memory_id) save_id_map(ID_MAP) # On search: memory_uuid = ID_MAP[faiss_pos] cur.execute("SELECT * FROM memories WHERE id = ?", (memory_uuid,)) 

4 — Time Decay on Resonance

Before: Resonance treated a memory from ten years ago identically to one from yesterday.

After: An exponential decay factor (half-life = 1 year by default) is multiplied into the resonance score. Old memories are still surfaceable — a floor of 0.3 prevents them disappearing entirely — but recent memories naturally score higher for the same semantic match.

decay = math.exp(-math.log(2) * age_days / half_life_days) resonance = intensity * similarity * text_richness * (0.3 + 0.7 * decay) 

5 — "Surprise Me" Command

Surfaces a memory you haven't thought about in a while by scoring memories on (1 - decay) × (1 / intensity) — favouring the old and quiet ones. It then samples probabilistically from the top 20%, so it isn't fully deterministic.

python soulcapsule.py surprise 

6 — Mood Drift Visualisation

Plots your emotional valence over time as a Unicode sparkline directly in the terminal. Each mood is mapped to a valence score (joy = +1.0, anger = −0.9, etc.), memories are bucketed into time slices, and the average valence per bucket becomes a bar in the sparkline.

python soulcapsule.py mood-drift --buckets 40
